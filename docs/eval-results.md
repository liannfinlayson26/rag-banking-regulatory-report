# Evaluation results — Baseline vs Agentic

Dataset: **HSBC Pillar 3 Disclosures at 31 December 2025 — baseline stress test**  
Generated by `eval.py` (measurement only — the agent, chunking and stores are unchanged). Baseline = `pillar3_baseline` (naive one-shot); Agentic = `pillar3_structured` (LangGraph self-check).

Each answer is scored deterministically on three axes by exact-number matching against gold (no LLM grader is allowed to pass a wrong number):
- **grounded** — every claim traceable to the provided context, with correct date/unit/basis. For computed answers (Q4 sum, Q10 delta), grounded = operands present in context **and** arithmetic correct (the computed result need not appear verbatim in a chunk).
- **numerically_correct** — right figure(s), right arithmetic.
- **complete** — every part of a multi-part question addressed.

> `rt` (grounded-retries) is the count of re-generations triggered by a failed grounded self-check (`grade_answer → generate`), which the graph guard caps at `MAX_RETRIES`. `gen` (generate calls, max **3**) is the *cumulative* count of every `generate` firing — it also includes rewrite-triggered regenerations, so it can exceed `MAX_RETRIES`; it is a cost proxy, not the retry bound. Only Q4 (compute+resolve) and Q8 (multi-hop rewrite for the Article-92 chunk) exercise the loops; the other eight settle in one pass (5 nodes). Max node firings = **17** (recursion limit 40).

## 3 · Grounding transparency

Not every pass is grounded the same way. So a reader knows exactly what kind of pass each is:

- **Q9 — inference-grounded (currency).** Q9's figure `$888,647m` is retrieval-grounded, but its **currency** (US dollars) is grounded by `$`/`$m`/`$bn` **inference**: the reporting-currency definition is front-matter (≈PDF p2), **outside** the extracted corpus (pages 4–27). This is recorded as an *acceptable* pass, flagged `currency_grounding="inference"` — not hidden as if it were retrieval-grounded.
- **Q4, Q10 — arithmetic-grounded (computed).** Q4 (`38,490 + 42,380 = 80,870 $m`) and Q10 (`120,716 − 106,472 = 14,244 $m`) are grounded on the **operands** being present in context plus **correct arithmetic**; the summed/delta result is *computed*, not quoted from a chunk.
- **All other questions — retrieval-grounded.** The stated figure is the correct one and appears in the retrieved context.
